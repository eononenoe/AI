# Claude Code 사용 가이드 (웹 버전 vs 명령어 버전)

## 1️ 개요
Claude Code는 **Anthropic의 Claude 모델**을 활용해
개발자가 **코드를 작성·리팩터링·분석·문서화**하도록 돕는 AI 도구입니다.

현재 사용 방식은 두 가지로 나뉩니다:
- **웹 버전 (https://claude.ai)**
- **명령어 버전 (Claude Code CLI)**

이 두 가지는 같은 Claude 모델을 사용하지만,  
**목적과 사용 환경이 다릅니다.**

---

## 2️ 웹 버전 (Claude Web)

###  개요
- Claude 공식 웹사이트(https://claude.ai)에서 사용하는 **대화형 UI 기반**.
- 마치 ChatGPT처럼 채팅창에 코드를 붙여넣거나 파일을 업로드하여 질문/요청 가능.

###  주요 특징
- `.py`, `.js`, `.ipynb` 등 다양한 코드 파일 업로드 가능  
- “이 함수 구조를 개선해줘”, “이 코드 버그 원인 찾아줘” 같은 대화형 요청
- 코드를 직접 실행하진 않음 (분석/제안 위주)

###  장점
- 직관적이고 접근성이 높음 (초보자도 쉽게 사용)
- 결과를 시각적으로 확인 가능
- 문서화나 코드 리뷰 요청 시 유용

###  단점
- 로컬 파일 구조나 프로젝트 전체 맥락은 한 번에 인식하기 어려움  
- 자동 파일 수정 불가 (복사/붙여넣기 수동 적용 필요)

---

## 3️ 명령어 버전 (Claude Code CLI)

###  개요
- 터미널 환경에서 Claude 모델과 직접 상호작용하는 방식.
- 로컬 프로젝트 폴더 안에서 바로 **파일을 생성·수정·리팩터링** 가능.

### 작동 방식
1. 프로젝트 루트에 **`claude.md`** 파일을 생성  
   → Claude에게 “이 프로젝트는 이렇게 다뤄라”는 기준을 제공
2. 명령어를 통해 Claude에게 지시:
   ```bash
   claude init             # claude.md 생성
   claude code explain .   # 코드 설명
   claude code edit main.py # 특정 파일 수정
   claude code add "로그인 API 추가"
  ```

3. Claude는 `claude.md`의 기준을 읽고, 실제 파일을 수정하거나 제안함.

### claude.md 예시

```markdown
# 프로젝트 개요
FastAPI 기반 내부 자동화 시스템

# 코드 스타일
- UTF-8 준수
- 주석은 영어로 작성
- 함수명은 동사로 시작

# 작업 요청
- src/routes/user.py 리팩터링
- test/user_test.py 테스트 케이스 보강
```
---

### 아키텍처 패턴(MVC 등)과 claude.md의 역할

Claude Code CLI는 단순히 코드를 수정하는 도구가 아니라,
**프로젝트의 구조적 기준(MVC, MVT, Clean Architecture 등)을 인식하고 따르는 방식**으로 작동합니다.
따라서 `claude.md` 안에 **우리 팀의 개발 구조**를 명확히 정의해두는 것이 핵심입니다.

#### 예시 1: Node.js + Express (MVC 구조)

```markdown
# Architecture
이 프로젝트는 MVC(Model-View-Controller) 패턴을 따릅니다.

- Model: /models (데이터 및 ORM)
- View: /views (EJS 템플릿)
- Controller: /controllers (비즈니스 로직 및 요청 처리)

Claude는 각 레이어의 역할을 혼합하지 않도록 수정합니다.
```

#### 예시 2: FastAPI (Clean Architecture)

```markdown
# Architecture
이 프로젝트는 Clean Architecture 형태로 구성됩니다.

- routes/: API 엔드포인트
- services/: 비즈니스 로직
- models/: 데이터 모델
- schemas/: Pydantic 데이터 검증

Claude는 비즈니스 로직을 routes에 작성하지 않고,
services 계층에 구현하도록 합니다.
```

 **이유:**

* Claude가 코드 구조를 이해하면, 자동 수정 시에도 레이어 책임이 깨지지 않음
* 여러 명이 Claude를 함께 써도 일관된 코드 스타일 유지 가능
* 프로젝트의 “개발 규칙서”로 활용 가능

---

### 장점

* 코드 베이스 전체를 실제로 다룰 수 있음 (자동 수정 가능)
* 팀 규칙(`claude.md`) 기반으로 일관된 코드 생성
* Git 워크플로와 자연스럽게 연동 가능

```

> “명령어 버전 설명 → claude.md 예시 → (이제 왜 claude.md가 중요한지: 아키텍처 구조 설명) → 장점”

---

###  장점

* 코드 베이스 전체를 실제로 다룰 수 있음 (자동 수정 가능)
* 팀 규칙(`claude.md`) 기반으로 일관된 코드 생성
* Git 워크플로와 자연스럽게 연동 가능

###  단점

* 초기 설정(API 키, CLI 설치 등) 필요
* 시각적 피드백은 적음 (결과는 파일 변경으로 확인)

##  두 버전의 차이와 선택 기준

| 구분     | 웹 버전         | 명령어 버전 (CLI)      |
| ------ | ------------ | ----------------- |
| 접근 방식  | 대화형 (UI)     | 명령어 기반 (터미널)      |
| 목적     | 코드 리뷰·설명·문서화 | 실제 코드 수정·생성       |
| 설정 필요  | 없음           | 있음 (설치 및 API 설정)  |
| 코드 수정  | 수동 (복사/붙여넣기) | 자동 (파일 직접 변경)     |
| 협업 적합성 | 개인 실험용       | 팀 단위, 규칙 기반 작업용   |
| 주요 파일  | 업로드한 코드 파일   | `claude.md` 기준 문서 |

---

##  왜 이렇게 구분되어 있는가?

* **웹 버전**은 “대화 중심 학습”을 위한 환경입니다.
  → 비개발자나 기획자도 AI와 코드 내용을 이해하고 협의 가능.

* **명령어 버전**은 “작업 자동화·개발 효율화”를 위한 환경입니다.
  → 실제 프로젝트 구조를 읽고, 파일을 자동으로 수정할 수 있음.

즉,

> * 웹 버전은 “아이디어를 테스트하거나 코드 리뷰 받을 때”,
> * 명령어 버전은 “실제 코드베이스를 다룰 때” 사용.

---

##  실무 적용 예시

### 🔹 이번 프로젝트 실험 (주말 실습 아이디어)

1. `claude.md` 작성 — 프로젝트 목표, 스타일, 요청 내용 기재
2. CLI 명령어로 Claude와 코드 리팩터링 진행
3. 웹 버전에서도 동일 요청을 보내서 결과 비교
4. 두 결과의 차이(정확도·맥락 이해·속도) 정리

→ 월요일 회의 때 “우리 팀 개발 환경에 맞는 사용 방향” 논의 가능.

---

##  결론

* Claude Code는 단순 AI 챗봇이 아니라, **개발 워크플로우 전체를 AI가 이해하도록 구조화한 시스템**이다.
* `claude.md`는 **팀의 개발 규칙과 방향성을 AI가 공유하도록 하는 핵심 파일**이다.
* 웹 버전은 이해 중심, 명령어 버전은 실행 중심이다.
* 두 버전을 병행하면, “대화로 이해하고, 명령어로 실행하는” AI 협업 프로세스가 완성된다.
